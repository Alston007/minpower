Solving Problems¶

solve.solve_multistage(power_system, times, datadir, solver='glpk', interval_hours=None, stage_hours=24, overlap_hours=0, problemfile=False, get_duals=True, showclock=True)[source]¶

Solve a rolling or multi-stage power systems optimization problem. Each stage will be one optimization run. A stage’s final conditions will be the next stage’s initial condition.

Parameters:
  • power_system – PowerSystem object
  • datadir – directory of spreadsheets
  • times – Timelist object. Will be split up into stages
  • interval_hours – number of hours per interval
  • stage_hours – number of hours per stage (e.g. run one commitment every stage_hours)
  • overlap_hours – number of hours that stages overlap (e.g. run a 36hr commitment every 24hrs)
  • writeproblem – save the formulation of each stage to a file
  • get_duals – get the duals, or prices, of the optimization problem
  • showclock – show the current system time at the start of each stage
Returns:

a list of Solution_UC objects (one per stage)
Returns:

a list of Timelist objects (one per stage)
solve.solve_problem(datadir='.', shell=True, problemfile=False, visualization=True, csv=True, solver='glpk', num_breakpoints=11, hours_commitment=24, hours_commitment_overlap=0, get_duals=True, dispatch_decommit_allowed=False, logging_level=20, logging_file=False, solution_file=False)[source]¶

Solve a optimization problem specified by spreadsheets in a directory. Read in data, create and solve the problem, and return solution. The problem type is determined by the data.

Parameters:
  • datadir – directory of data files, see get_data
  • shell – output solution information to the command line
  • problemfile – write the problem formulation to a problem-formulation.lp file
  • visualization – create a visualization of the solution and save it to a file
  • csv – create a spreadsheet of the solution
  • solver – choice of solver, a lowercase string
  • num_breakpoints – number of break points to use in linearization of bid (or cost) polynomials (equal to number of segments + 1)
  • hours_commitment – maximum length of a single unit commitment, times beyond this will be divided into multiple problems and solved in a rolling commitment
  • hours_commitment_overlap – overlap of commitments for rolling commitments
  • get_duals – get the duals, or prices, of the optimization problem
  • dispatch_decommit_allowed – allow de-commitment of units in an ED – useful for getting initial conditions for UCs
Returns:

Solution object
